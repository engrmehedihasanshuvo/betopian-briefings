Technology, Security & Governance

What protects the platform today, and what we will add in the SSO phase.

Implemented today (v1.0.0)

  • ERP login: email/password with JWT Bearer token + Odoo session cookie
  • Token refresh: automatic refresh on 401, forced logout on invalid session
  • Device binding: Android ID / iOS vendor ID sent at login
  • Mail security: IMAP/SMTP credentials in Flutter Secure Storage (OS encrypted)
  • Location guard: GPS required, blocking dialog if disabled, location cached for compliance
  • Version enforcement: Firestore APP_VERSION blocks outdated builds
  • HTTPS APIs: erp.betopiagroup.com, social-media API, mail.betopialimited.com
  • Separate API auth: Social feed uses dedicated Bearer + session headers
  • Remote secrets: OpenAI, Gemini, Pinecone keys loaded from Firestore (not hardcoded)
  • Architecture: Repository pattern, Provider state, bilingual EN/BN UI

Planned (SSO & Phase 2)

  • SSO / OIDC: single @betopialimited.com identity across all modules
  • Unified mail auth: no separate mail password after SSO
  • Central RBAC: role-based access from Admin Console
  • Certificate pinning: stricter TLS validation on production APIs
  • MFA / 2FA: optional second factor for sensitive HR actions
  • Biometric app lock: fingerprint/face unlock for app open
  • Audit logging: who accessed what, when, from which device
  • App attestation: Play Integrity / Apple DeviceCheck
  • Security review: penetration test before full 5,000-user rollout
  • GitLab token vault: secure dev portal credentials inside Betopian

Be transparent: mail still needs a second login today. SSO phase removes that and centralises control.
